Title: Creating a 3-Tier Application Using Docker
In this tutorial, we will walk you through the process of creating a 3-tier application using Docker. A 3-tier application architecture consists of a presentation tier, an application tier, and a data tier. We will use Docker to containerize each tier and orchestrate them to work together seamlessly.
Step 1: Push Images to Docker Hub Before we begin, make sure you have Docker installed on your machine. You can download Docker from the official website and follow the installation instructions for your operating system.
Firstly, pull the MongoDB and MongoDB Express images from Docker Hub. These images will be used for the data tier of our application.
Step 2: Create Docker Network Next, create a Docker network named mongo-network. This network will allow the containers in our application to communicate with each other.
Step 3: Create MongoDB and MongoDB Express Containers Now, create containers for MongoDB and MongoDB Express, and connect them to the mongo-network. MongoDB will run on port 27017 and MongoDB Express on port 8081.

Step 4: Create Node.js Application Create a basic Node.js application with a server.js file that will serve as the application tier. This file will connect to the MongoDB database.


Step 5: Dockerize the Node.js Application Create a Dockerfile for the Node.js application to containerize it. This Dockerfile will use the node image as the base image, copy the application files, install dependencies, expose port 4000, and start the server.
Build the Docker image using the following command: docker build -t 21bcp216img .

Step 6: Run the Node.js Application Container Finally, run the Node.js application container and connect it to the app-network. The application will be accessible at http://localhost:4000.
Conclusion: Congratulations! You have successfully created a 3-tier application using Docker. The presentation tier is represented by the Node.js application, the application tier by the MongoDB database, and the data tier by MongoDB Express. Docker allows you to easily containerize and orchestrate your application, making deployment and management much simpler.
